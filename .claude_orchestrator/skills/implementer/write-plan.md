# write-plan

implementer 用 skill です。

## 目的

実装前に変更方針を整理し、むやみに作業を広げないこと。

## 手順

1. task の目的を短く言い換える
2. 変更対象ファイルを洗い出す
3. 変更しないファイルや範囲も明確にする
4. 実装方針を箇条書きで整理する
5. テストや確認方法を整理する
6. 不明点があれば risks / questions に残す

## 期待される効果

- いきなり実装して構造を壊すことを防ぐ
- 変更対象の見落としを減らす
- reviewer / director が判断しやすくなる
