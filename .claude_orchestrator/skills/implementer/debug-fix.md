# debug-fix

## 目的

既存実装の不具合や軽微な挙動不整合を、最小変更で安全に修正する。

## 基本方針

- まず再現条件を確認する
- 原因箇所を絞る
- 必要最小限の修正に留める
- 既存の正常系を壊さない
- 修正後の影響範囲を確認する
- 過剰なリファクタは行わない

## 手順

1. 不具合の症状を整理する
2. 関連ファイルと関連関数を特定する
3. 原因を絞る
4. 最小差分で修正する
5. 修正後に同じ症状が解消したか確認する
6. 既存フローに副作用がないか確認する

## 注意

- bugfix task であっても、スコープ外の改善は混ぜない
- UI不整合修正では表示だけでなく状態整合も確認する
- 長時間プロセスを残す確認方法は避ける
- GUI確認が必要な場合は確認後すぐ終了する
