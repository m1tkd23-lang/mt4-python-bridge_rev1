# migration-safety-check

## 目的

データ構造変更（モデル・JSON・DB等）において、
既存データを壊さず安全に移行できるようにする。

## 基本方針

- 既存データとの後方互換を最優先する
- 新規フィールドはデフォルト値で補完する
- 既存構造を前提に読み込み可能にする
- 破壊的変更は避ける

## 手順

1. 既存データ構造を確認する
2. 新しいフィールドを定義する
3. from_dict / load 処理でデフォルト補完を実装する
4. save 時に新構造で保存されることを確認する
5. 旧データ → 新コードで読み込めるか確認する

## チェックポイント

- 既存 tasks.json がそのまま読み込めるか
- 新フィールドが None / 空値で安全に扱えるか
- save 後にデータが壊れないか
- UI 側が未対応フィールドでクラッシュしないか

## 注意

- フィールド削除や名称変更は極力避ける
- 必要なら段階的移行（deprecated → removal）を行う
- 今タスクでは migration のみ行い、UI変更は最小限にする
