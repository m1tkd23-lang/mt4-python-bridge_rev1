# 複数月評価フロー方針

## 目的

Exploration GUI の評価フローを単一CSV中心から複数月安定性評価中心に転換する。
単月最適化ではなく、複数月にわたって安定する条件を探すことが本来の主目的である（開発の目的本筋 セクション2「全月を通した安定性を重視」に準拠）。

---

## 現状の課題

現在の Exploration GUI は以下の構成になっている。

* `input_panel.py`: CSV File（単一CSV）を主入力とし、CSV Dir は任意入力
* `main_window.py`: `BollingerLoopConfig` に `csv_path`（必須）と `csv_dir`（任意）を渡す
* `exploration_loop.py`: `csv_path` で単月バックテスト、`csv_dir` 指定時に全月横断評価を追加実行
* `refinement.py`: 探索結果のスコアリング・refine は単一実行結果ベース

問題点:

1. デフォルトの探索フローが単一CSVに偏っており、複数月安定性が主軸になっていない
2. 「直近3ヶ月で探索し、全期間で確認する」という2段階フローが存在しない
3. refine が単月結果ベースであり、複数月集約結果の上位候補を使う設計になっていない
4. CSV ファイル選択が手動であり、直近N ヶ月の自動選択がない

---

## GUI仕様: CSV選択モード

CSV Dir 内のCSVファイルを対象として、以下の3モードを GUI 上に導入する。

### Selected 3 months（デフォルト）

* CSV Dir 内の全 CSV をファイル名でソートし、直近3ヶ月を自動選択する
* 現在のデータ構成例: `USDJPY-cd5_20250521_2025-05.csv` 〜 `USDJPY-cd5_20250521_2026-04.csv`（12ヶ月）
* デフォルトでは直近3件（2026-02, 2026-03, 2026-04）を探索対象とする
* 探索速度と安定性のバランスを取るための標準モード
* csv_path は選択された3ヶ月のうち最新のCSVを使用する

### All CSVs in folder

* CSV Dir 内の全 CSV を探索対象とする
* 探索結果の最終確認用として使う（全期間で安定するか検証）
* 探索段階では時間がかかるため、良い候補が見つかった後の確認フェーズで使用する

### Custom selection

* CSV Dir 内の CSV をチェックリストで個別に選択する
* 特定月の除外や、特定の期間だけを対象にした探索が可能
* 上級者向けオプション

---

## 探索フローの2段階構成

### Phase 1: 探索（Selected 3 months）

* 目的: 直近3ヶ月で安定する条件を効率的に見つける
* 使用モード: Selected 3 months（デフォルト）
* 評価対象: 選択された3ヶ月の集約結果（aggregate_monthly_stats → evaluate_integrated）
* refine: 複数月集約結果の上位候補を使って次回探索範囲を生成

### Phase 2: 確認（All CSVs in folder）

* 目的: Phase 1 で見つかった良い条件がフォルダ内全CSVでも安定するか検証
* 使用モード: All CSVs in folder
* 探索は行わない。Phase 1 の上位候補パラメータをそのまま全期間で再評価する
* 全期間で安定していれば採択候補として確定する

## multi-month 評価方針

### 探索時の評価対象変更

現状の `run_bollinger_exploration()` は csv_path で単月評価 + csv_dir で全月横断評価を行っているが、
複数月評価フローでは以下のように変更する。

* **選択された複数月CSVのみを横断評価の対象とする**（csv_dir 内の全CSVではなく、選択されたCSVリスト）
* `BollingerLoopConfig` / `BollingerExplorationConfig` に csv_paths: list[str] を追加し、複数CSVを直接指定可能にする
* csv_paths が指定された場合、csv_dir よりも csv_paths を優先する
* 単月評価用の csv_path は csv_paths の最後の要素（最新CSV）を使用し、横断評価は csv_paths 全体で行う

### 集約評価の継続利用

* `aggregate_monthly_stats()` → `evaluate_integrated()` の既存パイプラインはそのまま活用する
* 3ヶ月評価でも12ヶ月評価でも同じ `IntegratedThresholds` を使用する
* ただし、3ヶ月評価の場合は `min_avg_pips_per_month` の閾値を月数に応じて調整する可能性がある（後続検討）

## refine 方針の変更

### 現状

`refinement.py` の `build_refinement_plan()` は `BollingerExplorationResult` のリストを受け取り、スコアリングを行う。
スコアリングの重み配分:

* cross_month_score: 0.40（最大ウエイト）
* pips_score: 0.30
* pf_score: 0.20
* verdict_score: 0.07
* win_rate_score: 0.03

### 変更方針

1. **refine は複数月集約結果の上位候補を使う**: 単月結果ではなく、選択された複数月の集約評価結果（integrated_evaluation）をベースにスコアリングする。現在のスコアリングの `cross_month_score` ウエイト（0.40）はそのまま活用できる

2. **単一最適値ではなく次回探索範囲を作る**: 現在の `summarize_parameter_trend()` → `generate_refined_range()` のパイプラインは既にこの方針を実現している。top_results の上位候補から weighted_median を計算し、shrink_ratio に基づいて探索範囲を縮小する方式を継続する

3. **実在した良結果の組み合わせを seed として残す**: 現在の `seed_overrides_list` の仕組みがこれに該当する。`build_refinement_plan()` が top_results から seed 候補を抽出し、次回ループの `seed_overrides_list` に渡す方式を継続する

### 既存 refinement.py との整合性

現在の `refinement.py` は上記の方針に概ね対応済みである。主な変更点は:

* スコアリング入力が複数月集約結果を含む `BollingerExplorationResult` であること（既に `cross_month_evaluation` / `integrated_evaluation` フィールドを持つ）
* `score_cross_month()` が `integrated_evaluation` を優先参照すること（既に実装済み）
* seed_overrides_list の生成ロジック（既に実装済み）

## 段階導入方針

### Step 1: CSV選択モードの導入と csv_paths 対応

* input_panel に CSV選択モード（RadioButton / ComboBox）を追加
* CSV Dir 指定時に直近3ヶ月を自動検出するロジックを実装
* BollingerLoopConfig に csv_paths: list[str] | None フィールドを追加
* exploration_loop の run_bollinger_exploration() で csv_paths を csv_dir より優先する分岐を追加
* main_window で選択モードに応じた csv_paths を組み立てて config に渡す

### Step 2: 2段階フローの明示化

* GUI に Phase 表示（探索中 / 確認中）を追加
* Phase 1 完了後に「全期間で確認する」ボタンまたは自動遷移を提供
* result_panel に月別内訳の表示を追加

### Step 3: refine の複数月集約ベース強化

* refine 時に Phase 1 結果（3ヶ月集約）と Phase 2 結果（全期間集約）を区別して表示
* 全期間確認結果に基づく最終採択判定の支援表示を追加

## csv_path / csv_paths / csv_dir フィールド定義と優先順位

### 3フィールドの役割

| フィールド | 型 | 役割 |
|---|---|---|
| csv_path | str（必須） | 単月評価用の基準CSV。csv_paths 未指定時の単独評価に使用 |
| csv_paths | list[str] \| None（任意） | 複数月横断評価用のCSVリスト。GUI選択モードで組み立てられる |
| csv_dir | str \| None（任意） | CSVディレクトリ。csv_paths 未指定時のフォールバックとして全CSV横断評価に使用 |

### 優先順位ロジック（横断評価対象の決定）

```
csv_paths が指定されている場合:
  → csv_paths のリストで横断評価を実行
  → csv_path は csv_paths[-1]（最新CSV）を使用（単月評価用）

csv_paths が未指定 かつ csv_dir が指定されている場合:
  → csv_dir 内の全CSVで横断評価を実行（既存動作を維持）
  → csv_path はユーザ指定値をそのまま使用

csv_paths も csv_dir も未指定の場合:
  → csv_path のみで単月評価を実行（既存動作を維持）
```

### csv_path の自動決定ルール

csv_paths が指定されている場合、csv_path は **csv_paths の最後の要素（最新CSV）** を使用する。
理由: csv_paths はファイル名でソート済みの時系列リストであり、直近データでの単月評価が探索の基本単位として最も適切であるため。

GUI の Selected 3 months モードでは csv_paths = [2026-02, 2026-03, 2026-04] のとき csv_path = 2026-04 のCSVとなる。

### BollingerLoopConfig / BollingerExplorationConfig への適用

* csv_paths: list[str] | None フィールドを新規追加（Step 1 実装タスクで対応）
* csv_path（既存・必須）と csv_dir（既存・任意）は後方互換のため残す
* run_bollinger_exploration() 内の横断評価ロジックで、csv_paths → csv_dir → csv_path のみ の順に分岐する

## 既存コードとの整合性確認

### exploration_loop.py との整合

* `BollingerExplorationConfig` は既に `csv_dir` フィールドを持つ。csv_paths 追加は既存フィールドと共存させ、csv_paths が指定されていれば csv_dir より優先する
* `run_bollinger_exploration()` 内の全月横断評価ロジック（L532-L566）は csv_dir 内の全CSV を対象としている。csv_paths 対応では、このロジックを csv_paths リストを使うように分岐する
* `BollingerLoopConfig` の既存フィールド（csv_path, csv_dir）は互換性のため残す
* LoopConfig.timeframe のデフォルト値 "M5" は変更しない（TASK-0042 memory より破壊的変更を避ける）

### refinement.py との整合

* `score_cross_month()` は既に `integrated_evaluation` を優先参照する設計（L198-L200）
* `build_refinement_plan()` の入力は `BollingerExplorationResult` リストであり、各要素が `integrated_evaluation` を持っていれば複数月集約ベースのスコアリングが自動的に行われる
* seed_overrides_list の生成（L112-L128）は既に実装済み

### input_panel.py との整合

* 現在の CSV File / CSV Dir の2入力は残す（後方互換）
* CSV選択モードは CSV Dir が指定されている場合にのみ有効化する
* CSV Dir 未指定時は従来どおり CSV File 単体での探索となる
