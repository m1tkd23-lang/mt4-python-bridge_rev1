## planner の判断ルール
- 今回完了した task の延長線上で、次にやる価値が高い候補を提案してください
- 候補は 1〜3 件までにしてください
- 抽象論ではなく、そのまま task 化しやすい粒度で提案してください
- 候補ごとに title, description, priority, reason, context_files, constraints を必ず埋めてください
- 既存 task と重複しそうな案や、価値の低い案は避けてください
- docs と source task の内容に整合する案を優先してください
- 実在しないファイル名をむやみに作らないでください
- context_files は、分かる範囲で repo 内の具体的な相対パスを入れてください
- constraints は、その task を安全に実装するための制約を具体的に書いてください
- 候補が複数ある場合は priority で強弱を付けてください
- priority は high / medium / low のいずれかにしてください
- proposal_id は PLAN-0001, PLAN-0002, PLAN-0003 の形式で付けてください
- source_task_id は入力 task_id と一致させてください
- cycle は入力 cycle と一致させてください
- role は必ず "planner" にしてください

## 重要な実行指示
- planner report JSON を作成するだけでなく、必ず指定された保存先にファイルとして保存してください
- 保存先は必ずこの task の対象 repo 配下の .claude_orchestrator を使用してください
- 親repoや別repoの .claude_orchestrator に保存してはいけません
- ファイル保存が成功してから応答を返してください

## output rules
- JSON のみを返してください
- コードフェンスは禁止です
- 前置き文は禁止です
- proposals は 1〜3 件にしてください
- summary は source task の完了内容と次 task 提案の意図が分かるように簡潔に書いてください
